# Product Requirement Document (PRD) - 实时效果预览与即时下载

## 1. 目标
简化工作流，移除“生成”和“对比”步骤。用户在调整参数时即时看到最终效果（所见即所得），并可直接下载当前预览的图片。

## 2. 功能需求

### 2.1 交互流程
1.  **上传图片**：用户上传图片后，显示实时预览 Canvas。
2.  **实时堆叠预览**：
    - 预览区域即为最终结果区域。
    - 随着用户输入多行文本，Canvas 实时增高并绘制堆叠效果。
    - 所有样式调整（字体、颜色等）即时反映在 Canvas 上。
3.  **即时下载**：
    - 预览区域附近（或覆盖）提供“下载图片”按钮。
    - 点击即下载当前 Canvas 的内容。

### 2.2 界面变更
- **移除**：“生成场景”按钮 (`#generate-btn`)。
- **移除**：底部的结果对比区域 (`#preview-section`, `.comparison-container`)。
- **新增**：“下载结果”按钮，放置在预览区域内部（悬浮）或紧贴下方。

### 2.3 技术实现摘要
- **Canvas**：`#live-preview-canvas` 承载所有绘制逻辑（包含多行堆叠）。
- **下载逻辑**：直接将 `#live-preview-canvas` 转为 dataURL 进行下载。
- **性能**：由于 Canvas 尺寸可能较大（与原图一致），需确保重绘性能（防抖可能已足够）。
